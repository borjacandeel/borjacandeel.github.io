OSPF — Protocolo de Encaminamiento de Estado de Enlace

Open Shortest Path First: protocolo de enrutamiento dinámico de estado de enlace para redes IP.

A) Concepto

▾

¿Qué es? OSPF (Open Shortest Path First) es un protocolo de enrutamiento dinámico de estado de enlace. Cada router construye un mapa completo de la topología y calcula la ruta más corta (algoritmo de Dijkstra).

¿Por qué existe? Supera las limitaciones de RIP: no tiene límite de 15 saltos, converge más rápido, soporta VLSM/CIDR y es escalable para redes grandes.

¿Dónde va? Se configura en routers. Cada interfaz del router que pertenece a una red participante necesita declararse en el proceso OSPF con su área correspondiente.

B) Conceptos Clave

▾
TérminoQué significaPor qué importa
Process IDIdentificador local del proceso OSPF en el router (1-65535)No tiene que coincidir entre routers; es solo local
Router-IDIdentificador único del router en el dominio OSPF (formato IP)Evita conflictos; se recomienda configurarlo manualmente
ÁreaAgrupación lógica de routers. El Área 0 es el backbone obligatorioReduce el tamaño de la LSDB y el tráfico de actualización
Máscara WildcardInverso de la máscara de subred (ej. /24 → 0.0.0.255)Se usa en el comando network para indicar qué interfaces participan
DR/BDRDesignated Router / Backup DR elegido en redes multiaccesoReduce la cantidad de adyacencias OSPF en segmentos Ethernet
CosteMétrica OSPF = 100Mbps / ancho de banda de la interfazDetermina qué rutas prefiere OSPF; FastEthernet = 1, Serial = 64
LSALink State Advertisement: anuncio de estado de enlace que cada router inundaCon ellos cada router construye la LSDB (base de datos topológica)
LSDBLink State Database: base de datos de la topología completaDebe ser idéntica en todos los routers de la misma área

VLANs — Redes de Área Local Virtuales

Segmentación lógica de la red en el nivel 2 para aislar tráfico y mejorar la seguridad.

A) Concepto

▾

¿Qué es? Una VLAN (Virtual Local Area Network) es una red lógica que agrupa puertos de uno o varios switches sin importar su ubicación física, creando dominios de difusión separados.

¿Por qué existe? Permite segmentar la red (p. ej., personal/invitados/servidores) mejorando la seguridad y reduciendo el tráfico de broadcast, sin necesidad de hardware adicional.

¿Dónde va? Las VLANs se configuran en switches. El enrutamiento entre VLANs se hace en el router mediante subinterfaces (Router-on-a-Stick) o directamente en un switch de capa 3.

B) Conceptos Clave

▾
TérminoQué significaPor qué importa
Puerto de AccesoPuerto que pertenece a una sola VLAN; conecta dispositivos finales (PCs)El tráfico no lleva etiqueta 802.1Q hacia el dispositivo final
Puerto TroncalEnlace que transporta varias VLANs entre switches o switch-routerUsa etiquetas 802.1Q para distinguir a qué VLAN pertenece cada trama
VLAN NativaVLAN sin etiquetar en un enlace troncal (por defecto VLAN 1)Debe coincidir en ambos extremos del trunk o habrá pérdida de tráfico
Router-on-a-StickRouter con una sola interfaz física dividida en subinterfaces, una por VLANPermite enrutar entre VLANs sin un switch de capa 3
SubinterfazInterfaz lógica sobre una interfaz física (ej. Gig0/0.10 para VLAN 10)Cada subinterfaz actúa como puerta de enlace de una VLAN
encapsulation dot1QEtiquetado 802.1Q de la subinterfaz para una VLAN específicaSin esto el router no sabe a qué VLAN pertenece el tráfico entrante

Enrutamiento Estático

Configuración manual de rutas para redes pequeñas o topologías simples y predecibles.

A) Concepto

▾

¿Qué es? El enrutamiento estático consiste en configurar manualmente en cada router las rutas hacia las redes remotas, indicando la dirección de red destino, la máscara y el siguiente salto (o interfaz de salida).

¿Por qué existe? No consume ancho de banda en actualizaciones, es predecible y seguro para topologías simples. Ideal en entornos pequeños o como ruta por defecto (default route).

¿Dónde va? Se configura exclusivamente en routers. Los PCs solo necesitan una puerta de enlace; no configuran rutas estáticas.

B) Conceptos Clave

▾
TérminoQué significaPor qué importa
ip routeComando para crear una ruta estática: red destino + máscara + siguiente saltoEs la instrucción central del enrutamiento estático
Siguiente SaltoIP del router adyacente hacia la red destinoEl router envía los paquetes a esa IP para alcanzar la red remota
Ruta por DefectoRuta hacia 0.0.0.0/0: atrapa todo tráfico sin ruta más específicaSe usa para apuntar al ISP o al router de salida
Ruta FlotanteRuta estática con distancia administrativa alta (>1) como respaldoSolo se usa si la ruta principal falla
Distancia Admin.Valor que prioriza rutas frente a otras (estática=1, RIP=120, OSPF=110)Menor distancia = mayor preferencia

RIP — Protocolo de Información de Enrutamiento

Protocolo de enrutamiento dinámico por vector de distancia, simple de configurar para redes pequeñas.

A) Concepto

▾

¿Qué es? RIP (Routing Information Protocol) es un protocolo de enrutamiento dinámico por vector de distancia. Usa el número de saltos como métrica (máximo 15; 16 = inalcanzable).

¿Por qué existe? Es el protocolo de enrutamiento más simple de configurar. RIPv2 añade soporte para VLSM/CIDR y autenticación. Útil para aprender conceptos y en redes muy pequeñas.

¿Dónde va? Exclusivamente en routers. Los routers intercambian tablas de enrutamiento cada 30 segundos por defecto.

B) Conceptos Clave

▾
TérminoQué significaPor qué importa
Métrica (Saltos)Número de routers que atraviesa un paquete hasta el destinoRIP elige la ruta con menos saltos, sin importar el ancho de banda
RIPv1 vs RIPv2v1: classful (no envía máscara); v2: classless (envía máscara con la red)Siempre usar v2 en topologías con VLSM o subredes no classful
no auto-summaryDesactiva la sumarización automática de rutas a classful¡Obligatorio en RIPv2 con subredes! Sin él, las rutas se resumen y se pierden
passive-interfaceInterfaz que no envía actualizaciones RIP (solo recibe)Se usa en interfaces hacia PCs para no desperdiciar tráfico
network (classful)En RIP, la red declarada en "network" debe ser la dirección de clase (A/B/C)RIP participa en todas las interfaces que pertenezcan a esa red classful

ACLs, VTP, Serial e Interfaces — Extras de clase

Contenido extraído de las prácticas: listas de control de acceso, VTP, interfaces Serial y Loopback.

ACL Estándar — Concepto

▾

¿Qué es? Una ACL (Access Control List) es una lista de reglas que el router evalúa secuencialmente para permitir o denegar tráfico. Las ACL estándar filtran solo por dirección IP de origen.

¿Por qué existe? Controla qué hosts o redes pueden acceder a determinados recursos, implementando políticas de seguridad sin hardware adicional.

¿Dónde va? Se configura en routers. Se aplica a una interfaz en dirección in (entrante) o out (saliente). Regla de oro: ACL estándar lo más cerca posible del destino.

TérminoQué significaPor qué importa
permit / denyAcción de la regla: dejar pasar o bloquearEs lo que hace la ACE (entrada de la ACL)
implicit deny anyAl final de toda ACL hay un deny implícito que bloquea todoSi no añades un permit any, bloqueas todo lo que no coincide
ACL numeradaSe identifica por un número (1-99 estándar, 100-199 extendida)Más rápida de escribir, no se puede editar línea a línea
ACL con nombreSe identifica por un nombre descriptivoSe puede editar (añadir/borrar líneas) sin borrar toda la ACL
ip access-groupAplica la ACL a una interfaz en dirección in/outSin aplicarla, la ACL no filtra nada
WildcardIgual que en OSPF: inverso de la máscaraIndica qué bits del IP se comparan

C) Referencia de Comandos ACL

▾
ACL Estándar Numerada
ROUTER — Crea regla para denegar una red completa (wildcard 0.0.0.255 = /24) 
access-list 1 deny 192.168.11.0 0.0.0.255
access-list 1 permit any   ← sin esto, el deny implícito bloquea TODO
ROUTER
⚠️
Sin ip access-group la ACL no hace nada. Aplícala siempre en la interfaz correcta y con la dirección in/out adecuada.
— Aplica la ACL 1 en la interfaz de salida (out) hacia el destino a proteger 
interface GigabitEthernet0/0

VTP — VLAN Trunking Protocol

▾

¿Qué es? VTP permite que un switch servidor (Server) distribuya automáticamente la base de datos de VLANs al resto de switches clientes (Client) a través de los enlaces trunk.

¿Por qué existe? Sin VTP hay que crear las VLANs manualmente en cada switch. Con VTP basta con crearlas en el servidor y se propagan solas.

Modos: Server (crea/modifica/elimina VLANs y las propaga) · Client (recibe y aplica cambios, no puede crear VLANs) · Transparent (no participa en VTP, reenvía mensajes).

Switch Servidor VTP (normalmente S1_sw1)
SWITCH
⚠️
El dominio VTP debe ser IDÉNTICO en todos los switches. Distingue mayúsculas. Un dominio incorrecto impide la propagación.
— Configura el switch como servidor VTP con dominio y contraseña
vtp mode server
vtp domain EMPRESA
vtp password cisco123
vtp version 2
SWITCH— Crea las VLANs en el servidor (se propagan automáticamente a los clientes)
vlan 30
 name ADMINISTRACION_A
vlan 40
 name INFORMATICA_A
vlan 50
 name RRHH_A
Switches Cliente VTP (S1_sw2, S1_sw3, S1_sw4…)
SWITCH
⚠️
El trunk debe estar configurado ANTES de que el cliente pueda recibir las VLANs del servidor.
— Configura el switch como cliente VTP (recibe VLANs del servidor automáticamente)
vtp mode client
vtp domain EMPRESA
vtp password cisco123
vtp version 2

! Trunk hacia el switch servidor (obligatorio para recibir VLANs)
interface GigabitEthernet0/1
 switchport mode trunk

Interfaces Serial y Loopback

▾

Serial: Enlace WAN punto a punto entre routers. Uno de los extremos es DCE (Data Communications Equipment) y debe configurarse con clock rate. El otro es DTE y no necesita clock rate.

Loopback: Interfaz virtual siempre activa (never down). Se usa como Router-ID en OSPF, para pruebas de conectividad o para simular una red adicional.

Interfaz Serial (enlace WAN entre routers)
ROUTER
⚠️
clock rate solo en el extremo DCE. En Packet Tracer puedes ver qué extremo es DCE con: show controllers Serial0/0/0
— Configura el extremo DCE del enlace serial (el que tiene el reloj)
interface Serial0/0/0
 ip address 10.0.0.1 255.255.255.252
 clock rate 64000     ← solo en el lado DCE
 no shutdown
ROUTER— Configura el extremo DTE (el otro extremo del serial — sin clock rate)
interface Serial0/0/0
 ip address 10.0.0.2 255.255.255.252
 no shutdown   ← no lleva clock rate
ROUTER— Comprueba cuál es el extremo DCE/DTE de una interfaz serial
show controllers Serial0/0/0
Interfaz Loopback
ROUTER— Crea una interfaz loopback virtual (siempre activa, nunca cae) — útil para Router-ID en OSPF
interface Loopback0
 ip address 10.10.0.1 255.255.255.0  ← simula una red adicional o salida a Internet
ROUTER— Para incluir la loopback en OSPF (usa /32 porque es un host)

Gestión de configuración
ROUTERSWITCH— Guarda la configuración en NVRAM (persiste tras reinicio)
copy running-config startup-config
! Alternativa equivalente:
write memory
ROUTERSWITCH— Muestra la configuración activa completa
show running-config
ROUTER
⚠️
Muy recomendable en laboratorio: evita que el router intente resolver typos como nombres DNS, lo que congela la línea de comandos varios segundos.
— Desactiva la resolución DNS (evita esperas por typos en comandos)
no ip domain-lookup
ROUTER— Ver tablas de enrutamiento (C=conectada, S=estática, O=OSPF, R=RIP)
show ip route
ROUTERSWITCH— Resumen de estado de todas las interfaces: IP, estado físico y protocolo
show ip interface brief
